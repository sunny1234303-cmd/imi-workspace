# job-application 스킬

## 전체 개요

채용공고 텍스트를 받아 4단계로 맞춤 자소서를 완성한다. 각 단계마다 사용자 승인을 받고, 최종 승인 후에만 파일 저장.

```
Phase 1: 공고 파싱 + 회사 사업 분석
    └─ [체크포인트 1] 파싱 결과 확인
Phase 2: 직무요건 vs 내 경험 매핑 테이블
    └─ [체크포인트 2] 매핑 승인
Phase 3: 강점 우선순위 & 포지셔닝 전략
    └─ [체크포인트 3] 강점·방향 최종 확인
Phase 4: 자소서 완성본 + 이력서 조정 포인트
    └─ [체크포인트 4] "저장" 명령 후 파일 저장
```

**절대 규칙**
- 한 번에 한 Phase씩만 진행. 체크포인트를 건너뛰지 않는다.
- 파일 저장은 사용자가 명시적으로 "저장"이라고 말해야만 실행.
- 자소서의 모든 항목에 수치를 최소 1개 이상 포함.

## Phase 1: 공고 파싱 + 회사 사업 분석

### 1-1. 공고 텍스트 파싱

입력된 공고에서 다음 항목을 추출한다:
- 회사명
- 직무명 (포지션)
- 주요업무 (3~6개 불릿)
- 자격요건 (필수 항목들)
- 우대사항 (선호 항목들)
- 제출서류
- 채용 프로세스
- 근무지·고용형태·마감일 (공고에 있을 경우)

### 1-2. 회사 사업 분석

WebSearch를 사용해 아래 정보를 수집한다:
- 검색어 1: `[회사명] 회사 사업 서비스 소개`
- 검색어 2: `[회사명] 마케팅 최근 뉴스 2025 2026`
- 검색어 3 (필요 시): `[회사명] 채용 기업문화`

수집 후 아래 항목을 정리:
- **업종·산업군**
- **주요 제품·서비스**
- **타깃 고객** (B2C/B2B, 연령대, 세그먼트)
- **비즈니스 모델** (어떻게 돈을 버는가)
- **최근 마케팅·사업 동향** (최신 캠페인, 신규 서비스, 뉴스)
- **마케팅 팀이 현재 집중할 것으로 보이는 포인트**

## Phase 2: 직무요건 vs 내 경험 매핑 테이블

### 2-1. 매핑 생성 방법

경험 데이터베이스의 핵심 경험 4개 + 추가 경험을 기반으로 자격요건·우대사항 항목별 매핑:

**강도 기준**
- **★★★**: 직접 경험 + 수치 있음
- **★★**: 유관 경험 있음 (수치 약하거나 간접)
- **△**: 부분 경험 또는 재프레이밍 가능
- **✗**: 경험 없음 (솔직하게 표기)

**재프레이밍 규칙**
- △ 또는 ✗ 항목에는 "재프레이밍 방향" 메모 필수
- "도메인 미경험 → 구조적 역량이 업종을 가리지 않는다" 방향으로 접근
- 무리하게 있는 척 하지 않는다

### 2-2. [체크포인트 2] 출력 포맷

```markdown
## 직무요건 vs 내 경험 매핑

| 직무 요건 | 매핑되는 내 경험 | 강도 | 비고 |
|-----------|-----------------|------|------|
| [자격요건 항목] | [Case X: 구체적 경험] | ★★★ | - |
| [우대사항 항목] | [유관 경험] | ★★ | - |
| [약점 항목] | - | △ | 재프레이밍: "..." |
| [없는 항목] | - | ✗ | 커버 전략: "..." |

---

### 핵심 강점 요약 (★★★ 항목들)
1. ...
2. ...
3. ...

### 주요 약점 & 재프레이밍 전략
- **[약점]**: [재프레이밍 방향]
```

출력 후:
> "이 매핑 어떻게 생각해요? 빠진 경험이나 강도 수정할 게 있으면 알려줘요. 없으면 '다음'이라고 하면 강점 전략으로 넘어갈게요."

사용자가 경험 추가하거나 강도 조정하면 반영 후 재출력. "다음" → Phase 3.

## Phase 3: 강점 우선순위 & 포지셔닝 전략

### 3-1. 강점 선정 로직

Phase 2의 ★★★ 항목들 중 이 회사의 주요업무·자격요건과 가장 잘 맞는 3개 선정.
경험 DB의 포지셔닝 키워드(설계·자동화·전환)를 이 회사에 맞게 특화.

### 3-2. 포지셔닝 한 줄 설계

기본 포지셔닝 "마케팅 시스템을 설계하는 사람"을 이 회사용으로 변형:
- 리테일/이커머스: "데이터로 문제를 진단하고, 채널 구조를 바꾸는 마케터"
- 브랜드/콘텐츠: "브랜드의 결을 이해하고, 콘텐츠로 전환을 만드는 마케터"
- 스타트업/테크: "마케팅 자동화를 직접 설계하고 구축하는 마케터"
- CRM/퍼포먼스: "데이터 기반으로 채널을 설계하고 성과로 증명하는 마케터"

### 3-3. [체크포인트 3] 출력 포맷

```markdown
## 강점 우선순위 & 포지셔닝 전략

### 이 회사용 포지셔닝
> "[맞춤 한 줄]"

---

### 강점 Top 3

**1. [강점명]**
근거: [Case X] — [핵심 수치]
자소서에서 이렇게 풀기: [한 문장 각도]

**2. [강점명]**
근거: [Case X] — [핵심 수치]
자소서에서 이렇게 풀기: [한 문장 각도]

**3. [강점명]**
근거: [Case X] — [핵심 수치]
자소서에서 이렇게 풀기: [한 문장 각도]

---

### 약점 재프레이밍 전략
- ❌ [약점] → ✅ [자소서에서 이렇게 커버]

---

### 자소서 전체 흐름 (예고)
[지원동기 핵심 각도] → [강점1으로 직무역량 증명] → [강점2·3으로 구체화] → [포부]
```

출력 후:
> "이 방향으로 자소서를 쓸게요. 강점 순서나 포지셔닝 문구 바꾸고 싶은 게 있으면 말해줘요. 없으면 '다음'이라고 하면 자소서 작성할게요."

"다음" → Phase 4.

## Phase 4: 자소서 생성 + 이력서 조정 포인트

### 4-1. 자소서 작성 규칙

**공고에 질문 항목이 있는 경우:**
- 각 질문 항목별로 별도 작성
- 글자 수 제한이 명시되어 있으면 해당 제한 준수
- 각 항목에 수치 최소 1개 이상 포함
- Phase 3의 포지셔닝과 강점 우선순위 반영

**공고에 질문 항목이 없는 경우:**
- 3파트 범용 구조: [지원동기] + [직무역량] + [포부]
- 직무역량은 강점 Top 3 기반으로 소항목 구분해서 작성

**공통 규칙:**
- 수치는 반드시 맥락과 함께: "전년 대비 +47.4%" 가 아니라 "예산 없이 타깃을 재정의해 신규 고객 전년 대비 47.4% 증가"
- 브랜드 도메인 미경험 시 무리하게 연결 시도 금지. "구조적 역량이 업종을 가리지 않는다"는 방향 유지
- 각 항목 마지막에 이 회사에서 하고 싶은 것과 연결

### 4-2. 이력서 조정 포인트 생성

이 직무에 맞는 이력서 v2 (`17-만능-이력서-v2.html`) 수정 제안:
- 프로젝트 순서 재정렬 권장 (이 직무에서 가장 강한 Case 앞에)
- 커버/오프닝 문구 수정 제안
- 강조해야 할 항목 / 줄여도 되는 항목
- 추가하면 좋은 수치나 맥락

출력 후:

## 오류 처리

- **공고에 회사명이 없는 경우**: "회사명을 알려주세요."
- **WebSearch 결과 없음**: "회사 정보를 온라인에서 찾기 어렵네요. 공고 내 회사 소개 내용으로 분석할게요."
- **글자 수 초과 시**: 해당 항목 끝에 `(현재 X자 / 제한 Y자 — 추가 삭제 필요)` 표기
- **수치 부족한 항목**: 수치가 없으면 자소서 작성 전에 "이 경험에서 수치를 알 수 있으면 알려주세요" 요청
