***측정 ID만 복사하여 붙여넣는 것은 웹사이트 빌더(네스스,아임웹)나 CMS(Wordpress, Wix, Shopify 등)을 사용할 때에만 해당
**** 수동->GTM 옮겨심기 시 이전 코드를 둔 상태에서 GTM을 추가하면 페이지 뷰가 두배로 보임. 

데이터의 연속성을 유지하면서 안전하게 갈아타는 방법은 다음과 같습니다.
측정 ID 동일 시 데이터는 같은 보관함에 계속 쌓임.

1. GTM 컨테이너 생성 및 코드 설치: 먼저 GTM 코드를 웹사이트 <head>와 <body>에 삽입합니다. (이때 아직 GTM 내부에서 GA4 태그를 만들지 않았다면 데이터는 겹치지 않습니다.)
2. GTM 내 GA4 설정 완료: GTM 내부에서 측정 ID를 넣어 GA4 설정을 마칩니다. 하지만 아직 '제출(Publish)'은 하지 마세요.
3. 교체 작업 (가장 중요):
    * 기존에 수동으로 넣어두었던 GA4 수동 설치 코드를 삭제합니다.
    * 동시에 GTM의 설정을 '제출'하여 활성화합니다.
4. 확인: GA4의 실시간 보고서를 보면서 데이터가 한 번씩만 들어오는지 확인합니다.

- 데이터 실시간 연결 확인_활성 사용자 및 실시간 보고서(30분)
- 계정 액세스 관리 
Point : 기본 설정의 경우 ga4는 상세 데이터 보관 기간이 2개월 / 기간을 14개월로 늘려두어야 추후 YoY 분석 가능

맞춤 측정 기준 세팅 폼_
측정기준과 측정항목이 있어야 맞춤 이벤트 분석 가능

세 가지 옵션인 이벤트, 사용자, 상품 범위 측정기준 중에 선택합니다. 
선택한 범위에 따라 맞춤 측정기준이 데이터에 적용되는 방식이 결정됩니다.

- 이벤트 범위 측정기준 : 클릭/ 전환 같은 이벤트
- 사용자 범위 측정기준 : 지리적 위치
- 상품 범위 측정기준 : 색상

- 사용자 획득 / 트래픽 획득
: 사용자 획득 - 신규 유입 - 웹사이트나 앱에 도달한 사용자가 유입된 위치(신규유입, 브랜드 인지도)
: 트래픽 획득 - 세션 포함 - 재유입과 관련없이 유입을 기준으로 분석(현재 시점의 활동량)

Ex ) 월요일에 처음으로 브랜드를 알게 되어 검색을 통해 들어옴. 이후 인스타에서 광고를 보고 수요일에 들어옴
=> 사용자 획득 : 월요일 - 구글 / 수요일 - 구글
      트래픽 획득 : 월요일 - 구글 / 수요일 - 인스타
