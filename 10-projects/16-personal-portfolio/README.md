## 경력기술서 작성 원칙

### 금지 표현
1. **자기선언 금지** — "~역량이 있습니다", "~기여할 수 있습니다", "~할 수 있는 사람입니다" 등 스스로를 평가하는 문장
2. **논리적 주장 금지** — "~할수록 더 강력하게 작동합니다", "~배가됩니다" 등 근거 없는 미래 논리 서술
3. **고백·인정 표현 금지** — "인정합니다", "부족하지만", "아직 ~하지 못했지만" 등 약점을 스스로 부각하는 표현

### 작성 기준
- **사실과 수치만 서술한다** — 주장은 수치가 대신한다. 수치 없는 역량 주장은 쓰지 않는다.
- **경험이 말하게 한다** — 결론 문장 없이 경험과 결과를 나열하는 것만으로 충분하다.
- **마무리 문장을 경계한다** — "따라서 저는 적합한 인재입니다" 류의 마무리는 삭제한다.

### 올바른 예시
> ❌ "데이터 분석 역량을 바탕으로 NHR에 기여할 수 있습니다."
> ✅ "비수기 트래픽 하락을 단일 채널 의존 구조 문제로 진단하고 멀티채널 전환해 매출 203% 상승을 달성했습니다."

### 수치 사용 시 인과관계 오해 방지 원칙

수치 앞에 나열된 행동이 그 수치의 **직접 원인처럼 읽히지 않도록** 주의한다.

> ❌ "A/B 테스트 기반 소재 전환, 카카오 캐러셀 광고 운영으로 클릭률 +100%, ROAS +152%"
> — A/B 테스트·카카오 캐러셀만으로 달성한 것처럼 오해됨. 실제로는 검색광고·SEO·배너·CRM 통합 IMC 운영 전체의 결과임.

> ✅ "검색광고·SEO·CRM·배너를 통합한 IMC 운영으로 클릭률 +100%, ROAS +152% 개선"
> — 어떤 채널 구성의 결과인지 명시해 오독 방지.

**적용 원칙**
- 수치는 반드시 **어떤 범위(캠페인/채널/기간)의 결과인지** 명시
- 앞 문장의 행동과 뒤 수치 사이에 인과관계가 없으면 **문장을 분리하거나 배경을 앞에 서술**
- 복합 원인의 성과는 "~를 포함한 통합 운영 결과" 형태로 서술

### 사용 금지 수치

| 수치 | 사용 금지 이유 |
|------|--------------|
| 클릭률 +100% | 비교 기준(기존 클릭률)이 불명확해 맥락 없이 오해의 소지가 있음. 어떤 단위의 클릭률인지, 절대값이 어느 수준인지 설명 없이는 신뢰도 낮음. **이력서 전면 사용 금지.** |

## HTML 제출용 에세이 작성 시 고려사항

> `13-coloso-에세이.html` 작업 경험 기반 (2026-03-06)

### 폰트
- `@import url('https://cdn.jsdelivr.net/gh/orioncactus/pretendard/...')` — Pretendard CDN 사용
- 본문 폰트 기준: **12px** (16px 기본값은 PDF 1페이지 초과됨)
- `html { font-size: 12px; }` 로 root 설정 후 나머지 em 기반 조정

### PDF 출력 레이아웃
- `@media print` + `@page { size: A4; margin: 20mm 22mm; }` 필수
- 에세이 페이지 분리: `.essay + .essay { page-break-before: always; }` 사용
- `page-break-inside: avoid` — p 태그에 적용해 단락이 페이지 경계에서 잘리지 않도록

### 배경색·배경 그래픽 출력
- 브라우저 PDF 저장 시 배경색이 기본 제거됨
- 검정 박스, 번호 원형 등 배경이 있는 요소에 반드시 아래 속성 추가:
  ```css
  -webkit-print-color-adjust: exact;
  print-color-adjust: exact;
  ```
- 적용 대상: `.sub-section-label`, `.rule-list li::before`, `.lead` 등

### 구조 패턴
- 에세이 구분: `essay-number` (ESSAY 01) + `essay-title` + `lead` (강조 인용구) + `essay-body`
- 소챕터 구분: `.sub-section-label` — 검정 배경 흰 글씨 인라인 태그
- 번호 목록: CSS counter 기반 (`counter-reset` / `counter-increment`) — 검정 배경 숫자
